## 🚀 核心原则

### 一致性原则
- 前后端命名保持一致（camelCase）
- API 响应格式统一
- 错误码体系统一
- 日志格式统一

### 可读性原则
- 代码即文档
- 有意义的命名
- 适当的注释
- 清晰的结构

### 安全性原则
- 输入验证
- 敏感信息保护
- SQL 注入防护
- XSS 防护

### 性能原则
- 合理使用缓存
- 避免 N+1 查询
- 分页查询
- 异步处理

## 💡 常见问题

### Q: 规范太多，记不住怎么办？
A:
- 不需要全部记住，需要时查阅即可
- 重点记住核心原则和最佳实践
- 使用 IDE 插件和 Lint 工具自动检查
- AI 助手会自动遵循规范

### Q: 规范和实际情况冲突怎么办？
A:
- 优先遵循规范
- 如有特殊情况，需在 PR 中说明原因
- 如规范不合理，提出修改建议

### Q: 如何让 AI 遵循这些规范？
A:
- AI 会自动读取这些规范文档
- 使用技能书提供项目上下文
- 在 Prompt 中明确要求遵循规范
- Review AI 生成的代码

### Q: 前后端分离后如何保持一致性？
A:
- 使用统一的 API 接口规范
- 前后端都遵循命名规范
- 使用 Swagger/OpenAPI 自动生成类型
- 定期 Code Review
